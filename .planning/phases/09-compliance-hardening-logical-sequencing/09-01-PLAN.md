# Phase 9: Compliance Hardening & Logical Sequencing

## Objective
Enhance compliance and logical sequencing, specifically for forms and tables. Ensure form tab order follows logical reading order and implement validation skills for tables and forms. Update the PDF writer to set appropriate tab order flags and the tagging engine to respect form logical order. Support human-in-the-loop review for form tooltips.

## Tasks

### 1. Implement FormTabOrderSkill
- **File**: `src/pdf_accessibility/skills/remediation/forms.py`
- **Action**: Add `FormTabOrderSkill` which sorts form fields based on their geometric position (top-to-bottom, left-to-right) and updates their tab index if applicable, or ensures they are processed in this order during tagging.
- **Commit**: `feat(09-01): implement FormTabOrderSkill`

### 2. Implement Table Validation Skill (VALID-TBL-001)
- **File**: `src/pdf_accessibility/skills/validation/tables.py`
- **Action**: Create new file and implement `TableStructureSkill` to check for basic table structural integrity (e.g., at least one row, no empty tables).
- **Commit**: `feat(09-01): implement VALID-TBL-001 table validation`

### 3. Implement Form Validation Skill (VALID-FRM-001)
- **File**: `src/pdf_accessibility/skills/validation/forms.py`
- **Action**: Create new file and implement `FormFieldValidationSkill` to check if form fields have names and tooltips.
- **Commit**: `feat(09-01): implement VALID-FRM-001 form validation`

### 4. Update PdfWriterService for Tab Order
- **File**: `src/pdf_accessibility/services/pdf_writer.py`
- **Action**: Update `write_remediated_pdf` to set `/Tabs /S` (Structure order) on page dictionaries that contain form fields.
- **Commit**: `feat(09-01): set /Tabs /S on pages with forms in PdfWriterService`

### 5. Update TaggingEngine for Form Order
- **File**: `src/pdf_accessibility/services/tagging.py`
- **Action**: Ensure `TaggingEngine` tags forms in the order they appear in the `CanonicalPage.forms` list (which should be sorted by `FormTabOrderSkill`).
- **Commit**: `feat(09-01): tag forms in sorted logical order in TaggingEngine`

### 6. Update Review Model and Service for Form Tooltips
- **Files**: `src/pdf_accessibility/models/review.py`, `src/pdf_accessibility/services/review.py`
- **Action**: Add `tooltip` field to `ManualOverride` and update `ReviewService` to apply tooltip overrides to form fields.
- **Commit**: `feat(09-01): support form tooltip overrides in review models and service`

### 7. Update Review Dashboard UI for Form Tooltips
- **File**: `ui/src/app/review/[id]/page.tsx`
- **Action**: Add UI elements to edit form field tooltips in the Inspector sidebar.
- **Commit**: `feat(09-01): allow editing form tooltips in review dashboard UI`

### 8. Verification
- **Action**: Run existing tests and add new tests for form tab order and validation.
- **Commit**: `test(09-01): verify form tab order and validation skills`

## Success Criteria
- Form fields are sorted logically.
- `/Tabs /S` is set in the output PDF for pages with forms.
- Table and form validation skills are correctly registered and executed.
- Review dashboard allows manual correction of form tooltips.
- All tests pass.
